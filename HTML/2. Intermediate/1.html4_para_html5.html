Mudanças do HTML4 para o HTML5

Este documento explica, de forma contínua e didática, as principais diferenças entre HTML4 e HTML5. Serão apresentadas as razões das mudanças, exemplos práticos que você pode testar na sua IDE (por exemplo VS Code), e pequenas demonstrações de código que ilustram como migrar ou usar os recursos novos.

Estrutura do guia: 1 Doctype e estrutura; 2 Semântica; 3 Mídia nativa; 4 Formulários modernos; 5 Canvas, SVG e gráficos; 6 API e armazenamento; 7 Elementos e atributos depreciados; 8 Acessibilidade e boas práticas.

1. Doctype e estrutura básica

Em HTML4 o doctype era longo e específico, por exemplo: 

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN"
  "http://www.w3.org/TR/html4/loose.dtd">

Em HTML5 o doctype foi simplificado para uma única linha, tornando mais fácil começar um documento e reduzindo confusão: 

<!DOCTYPE html>

Por que isso importa? Porque o doctype declara o modo de renderização do navegador (standards mode) e a simplificação do HTML5 facilita a criação de arquivos sem erros de sintaxe ligados ao tipo de DTD.

Como testar na prática: crie um arquivo index.html com a linha <!DOCTYPE html> no topo, abra no navegador e veja que você já está em modo padrão. O VS Code e outros editores reconhecem automaticamente.

2. Semântica: novas tags estruturais

HTML5 introduziu várias tags semânticas que trazem significado ao documento (não apenas aparência). Em HTML4 costumávamos usar <div> com classes para separar seções. Em HTML5 usamos tags específicas:

  • <header> — cabeçalho da página ou de uma seção;
  • <nav> — bloco de navegação;
  • <main> — conteúdo principal;
  • <article> — conteúdo independente (post, notícia, artigo);
  • <section> — seção genérica do documento;
  • <aside> — conteúdo complementar ou lateral;
  • <footer> — rodapé da página ou seção.

Essas tags melhoram a leitura por desenvolvedores e por leitores de tela (acessibilidade), além de facilitar a indexação por mecanismos de busca. 

<header>
  <h1>Nome do site</h1>
</header>

<nav> ... </nav>

<main>
  <article> ... </article>
  <aside> ... </aside>
</main>

<footer> ... </footer>
Como fizemos este arquivo: no topo já usamos <header>, <nav>, <section> e <article>. Isso permite demonstrar aos alunos a semântica na prática. Se antes só havia divs genéricas, agora cada bloco tem significado próprio.

3. Mídia nativa: áudio e vídeo sem plugins

No HTML4, para inserir mídia muitas vezes era necessário usar plugins externos como Flash. HTML5 trouxe elementos nativos para áudio e vídeo: <audio> e <video>. 

<!-- Vídeo simples com legendas -->
<video controls width="640" poster="capa.jpg">
  <source src="meuvideo.mp4" type="video/mp4">
  <source src="meuvideo.webm" type="video/webm">
  <track kind="captions" srclang="pt-BR" src="legenda.vtt" default>
  Seu navegador nao suporta o elemento video.
</video>

Prática: Coloque um arquivo de vídeo na pasta do projeto e atualize o src. Use o atributo controls para mostrar a interface padrão (play, pause, volume). Para legendas use arquivo WebVTT com <track>.

Observação: nem todos os navegadores suportam todos os codecs, por isso ofereça mais de um <source> (mp4, webm).

4. Formulários modernos

HTML5 trouxe novos tipos de input e atributos que reduzem a dependência de JavaScript para validação básica. Exemplos incluem types: date, email, url, number, search, tel, range, color. 

<form action="#" method="post">
  <label for="email">E-mail</label>
  <input type="email" id="email" name="email" required>

  <label for="nasc">Data de nascimento</label>
  <input type="date" id="nasc" name="nasc">

  <label for="cor">Cor preferida</label>
  <input type="color" id="cor" name="cor">

  <button type="submit">Enviar</button>
</form>

Atributos úteis: required, placeholder, autofocus, pattern, min, max, step. Eles ajudam a validar dados já no navegador.

Como testar: abra a página em um navegador moderno e tente submeter o formulário vazio para ver o comportamento de required. Experimente inserir um e-mail inválido e note a validação nativa.

5. Canvas, SVG e gráficos

HTML5 trouxe a tag <canvas> para desenho 2D via JavaScript e também incentivou o uso de SVG inline. Antes era comum gerar gráficos do lado do servidor ou usar plugins; agora há APIs nativas. 

<!-- Canvas simples em HTML5 -->
<canvas id="meuCanvas" width="300" height="150">Seu navegador nao suporta canvas.</canvas>

<script>
  var c = document.getElementById('meuCanvas');
  var ctx = c.getContext('2d');
  ctx.fillStyle = '#0099cc';
  ctx.fillRect(10, 10, 100, 50);
</script>

Dica prática: peça aos alunos para desenhar formas simples e, depois, construir um pequeno gráfico de barras usando retângulos preenchidos.

6. APIs, armazenamento e novas possibilidades

HTML5 é muito mais que novas tags: traz APIs que permitem criar aplicações ricas no navegador:

  • localStorage / sessionStorage — armazenamento simples de chave/valor no navegador;
  • Geolocation API — obter localização do usuário (com permissão);
  • Fetch API — requisições HTTP modernas (substitui XMLHttpRequest em muitos casos);
  • Web Workers — threads de background para processamentos longos;
  • History API — controlar histórico sem recarregar páginas;
  • Drag and Drop, Drag events — suporte nativo a arrastar e soltar.
<!-- Exemplo simples localStorage -->
<script>
  // salvar
  localStorage.setItem('usuario', 'Lucas');
  // ler
  var nome = localStorage.getItem('usuario');
  console.log('Usuário salvo', nome);
</script>

Essas APIs aproximam o front-end do que antes só era possível com aplicações nativas ou plugins.

7. Elementos e atributos depreciados

HTML5 marcou elementos antigos como obsoletos e recomendou alternativas:

  • <font> — use CSS para tipografia;
  • <center> — use CSS (text-align:center);
  • atributos de apresentação como bgcolor, align, border em tabelas — prefira CSS;
  • Flash e outros plugins — substituídos por <video> e <audio> e por APIs modernas.
Migração prática: em vez de <font color="red">Texto</font>, faça: 
<span style="color:red">Texto</span>  /* ou melhor, via classe CSS */

8. Acessibilidade e SEO

HTML5 facilita melhores práticas de acessibilidade quando usamos marcação semântica correta (header, nav, main, article, aside, footer), atributos como alt em imagens, e o uso de aria-* quando necessário. Para SEO, motores de busca entendem melhor a estrutura semântica do documento. 

<img src="dr.jpg" alt="Dr. Silva atendendo paciente">

Use <h1> apenas uma vez por página para título principal, depois <h2>, <h3> etc.

Exemplo integrado prático

Abaixo segue um pequeno exemplo que junta: semântica, vídeo, formulário com input moderno e armazenamento local. Copie o bloco para um arquivo e teste no navegador. 

<!DOCTYPE html>
<html lang="pt-br">
<head><meta charset="utf-8"><title>Exemplo integrado</title></head>
<body>
  <header><h1>Clínica Exemplo</h1></header>

  <main>
    <article>
      <h2>Vídeo institucional</h2>
      <video controls width="480">
        <source src="video.mp4" type="video/mp4">
      </video>
    </article>

    <section>
      <h2>Contato rápido</h2>
      <form id="f">
        <label for="n">Nome</label>
        <input id="n" type="text" required>

        <label for="e">E-mail</label>
        <input id="e" type="email" required>

        <button type="submit">Enviar</button>
      </form>
    </section>
  </main>

  <script>
    // salva nome no localStorage quando o form for submetido
    document.getElementById('f').addEventListener('submit', function(ev){
      ev.preventDefault();
      var nome = document.getElementById('n').value;
      localStorage.setItem('nomeContato', nome);
      alert('Nome salvo localmente: ' + nome);
    });
  </script>
</body>
</html>
Teste esse exemplo substituindo video.mp4 por um arquivo de vídeo local. O formulário usa validação nativa e salva o nome no localStorage como demonstração da API.

9. Checklist de diferenças rápidas (resumo para memorização)

  • Doctype simplificado: <!DOCTYPE html>.
  • Tags semânticas adicionadas: header, nav, main, article, section, aside, footer.
  • Mídia nativa: <audio> e <video>.
  • Canvas e SVG para gráficos e desenhos.
  • Novos tipos de input e atributos para formulários.
  • APIs do navegador: localStorage, Geolocation, Fetch, Web Workers.
  • Elementos e atributos antigos depreciados: font, center, bgcolor, align.
  • Melhor suporte a acessibilidade e SEO com marcação semântica.
